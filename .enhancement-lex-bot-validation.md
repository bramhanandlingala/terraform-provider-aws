### Enhancement: Strengthen Input Validation for aws_lex_bot Data Source

This pull request improves the `aws_lex_bot` data source by adding input validation to the following arguments:
- `description`: Now enforces a maximum length of 200 characters.
- `idle_session_ttl_in_seconds`: Now enforces a value between 60 and 86400.
- `nlu_intent_confidence_threshold`: Now enforces a value between 0 and 1.

These changes align the data source's validation with the resource schema, preventing invalid values and surfacing errors earlier for users.

#### User Impact
- Invalid values for these arguments will now be rejected with clear error messages.
- No breaking changes for valid configurations.

#### Example Error
If a user provides an invalid value, such as `idle_session_ttl_in_seconds = 10`, they will now see:
> expected idle_session_ttl_in_seconds to be in the range (60 - 86400), got 10
